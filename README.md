# 项目任务管理系统

基于飞书多维表格的项目任务管理系统，支持任务管理、进度跟踪和智能报告生成。

## 功能特性

### 📋 任务管理
- **任务列表视图**: 清晰展示所有任务信息
- **项目看板视图**: 按项目分组展示任务进度
- **实时同步**: 与飞书多维表格实时同步数据
- **智能筛选**: 支持按项目、状态、紧急程度筛选
- **快速搜索**: 支持任务标题、项目、负责人搜索

### 🎯 任务字段
根据飞书表格结构设计：
- **任务事项**: 任务名称
- **所属项目**: 项目分类
- **对接人**: 任务负责人
- **截止日期**: 任务截止时间
- **创建时间**: 自动记录创建时间
- **是否已完成**: 完成状态
- **距离截止日**: 自动计算剩余天数

### 📊 智能报告
集成豆包大模型，自动生成：
- **日报**: 当日任务完成情况总结
- **周报**: 一周工作进展分析
- **项目报告**: 按项目维度生成专项报告
- **一键复制**: 支持报告内容快速复制

### 🎨 用户界面
- **响应式设计**: 支持桌面端和移动端
- **现代化UI**: 简洁美观的界面设计
- **交互友好**: 流畅的用户操作体验
- **状态指示**: 清晰的任务状态和优先级标识

## 技术架构

### 前端技术
- **HTML5**: 语义化标记

## 部署到Netlify

### 自动部署
1. 将代码推送到GitHub仓库
2. 在Netlify中连接GitHub仓库
3. 设置构建配置（已包含netlify.toml）

4. 部署完成

## 使用说明

### 首次使用
1. 打开系统，点击"同步数据"按钮
2. 系统将从飞书表格加载任务数据
3. 使用筛选和搜索功能查找任务

### 任务管理
- **新增任务**: 点击"新增任务"按钮
- **编辑任务**: 点击任务卡片进入编辑模式
- **状态更新**: 勾选"是否已完成"更新任务状态
- **批量操作**: 使用筛选功能批量查看特定任务

### 报告生成
1. 点击"生成报告"按钮
2. 选择报告类型（日报/周报）
3. 设置报告日期和项目筛选
4. 点击"生成报告"等待AI处理
5. 复制生成的报告内容

## 注意事项

### CORS代理
- 系统使用 `cors-anywhere.herokuapp.com` 作为CORS代理
- 首次使用需要访问代理网站激活服务
- 生产环境建议使用自建代理服务

### 数据安全
- 飞书表格设置为所有人可编辑
- API密钥已在代码中配置
- 建议定期更新访问凭证

### 性能优化
- 任务数据在本地缓存
- 支持增量同步更新
- 图片和静态资源CDN加速

## 故障排除

### 常见问题
1. **数据加载失败**: 检查网络连接和CORS代理状态
2. **报告生成失败**: 确认豆包API配置正确
3. **任务保存失败**: 检查必填字段是否完整
4. **界面显示异常**: 清除浏览器缓存重新加载

### 调试方法
- 打开浏览器开发者工具
- 查看Console面板的错误信息
- 检查Network面板的API请求状态
- 确认飞书表格访问权限
